Introduction: What is an API?

An API (Application Programming Interface) is like a waiter at a restaurant. You tell it what you want (request), it tells the kitchen (server), and then brings back your order (response). In short: it allows two different software systems to communicate with each other.

Key points about APIs:

  • They define how requests and responses should be structured.
  • Can be public (open for anyone) or private (restricted to your app).
  • Can be synchronous (wait for response) or asynchronous (process in background).

How APIs Work

The life cycle of an API request is simple:

  1. Client sends a request (GET, POST, PUT, DELETE) to the API endpoint.
  2. API receives the request and passes it to the server logic.
  3. Server processes the request (reads/writes database, performs calculations, etc.).
  4. Server sends a response back to the client in JSON or XML format.
  5. Client receives the response and shows the data to the user.

Student analogy: You request study notes from a friend via WhatsApp (request), your friend finds them in their folder (server), and sends them back (response).

How to Identify Efficient APIs

Not all APIs are created equal. An efficient API is:

  • Fast: Low response time is crucial (like instant WhatsApp replies).
  • Reliable: Handles requests consistently without crashing.
  • Scalable: Can handle more requests as your app grows.
  • Well-Documented: Easy for developers to understand and use.
  • Secure: Protects sensitive data (authentication, authorization).

You can measure efficiency using tools like Postman, curl, or automated performance tests.

REST API Principles

  • Stateless: Each request contains all the information; server doesn't store client session.
  • Resource-Based: APIs focus on resources like "students", "books", etc.
  • HTTP Methods: GET (read), POST (create), PUT (update), DELETE (delete).
  • Uniform Interface: Standardized URL patterns and responses.

2. Creating a Basic Server

Create a server.js file and write a simple server:

// server.js
const express = require('express');
const app = express();
const PORT = 3000;

// Middleware
app.use(express.json());

// Basic route
app.get('/', (req, res) => {
    res.send('Hello, Express API!');
});

app.listen(PORT, () => {
    console.log(`Server running on http://localhost:${PORT}`);
});

3. REST API Routes Example

Letâ€™s create CRUD operations for a "students" resource:

// server.js
let students = [{ id: 1, name: 'Alice' }];

// GET all students
app.get('/students', (req, res) => res.json(students));

// POST a new student
app.post('/students', (req, res) => {
    const newStudent = { id: Date.now(), ...req.body };
    students.push(newStudent);
    res.status(201).json(newStudent);
});

// PUT update a student
app.put('/students/:id', (req, res) => {
    const id = parseInt(req.params.id);
    students = students.map(s => s.id === id ? { ...s, ...req.body } : s);
    res.json({ message: 'Student updated' });
});

// DELETE a student
app.delete('/students/:id', (req, res) => {
    const id = parseInt(req.params.id);
    students = students.filter(s => s.id !== id);
    res.json({ message: 'Student deleted' });
});

Understanding CRUD Workflow

CRUD stands for Create, Read, Update, Delete. Every REST API revolves around this workflow:

  • Create: Add new data (POST).
  • Read: Fetch existing data (GET).
  • Update: Modify existing data (PUT/PATCH).
  • Delete: Remove data (DELETE).

Text-based CRUD workflow flowchart:

  v
[Request] --POST--> [Create new record]
  |
  --GET--> [Fetch data]
  |
  --PUT--> [Update data]
  |
  --DELETE--> [Delete record]
  |
  v
[Response]

4. Middleware in Express

Middleware functions act like teaching assistants: they process requests before they reach the route handler.

// Simple logging middleware
app.use((req, res, next) => {
    console.log(`${req.method} ${req.url}`);
    next();
});

Other examples include authentication checks, error handling, and rate limiting. Think of it as security or processing helpers.

5. Advanced Topics

  • Routing with Express Router: Organize routes into separate modules for clarity.
  • Error Handling Middleware: Capture and manage errors without crashing your app.
  • Connecting to Databases: Use MongoDB, PostgreSQL, or MySQL for persistent storage.
  • Authentication & Authorization: JWT, OAuth, or session-based security.
  • Async/Await: Efficiently handle asynchronous operations and avoid callback hell.
  • Environment Variables: Store secrets (like API keys) safely using .env files.
  • Testing APIs: Use tools like Jest, Mocha, or Supertest for automated API testing.
