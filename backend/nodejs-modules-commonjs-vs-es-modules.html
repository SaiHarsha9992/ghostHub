Node.js Modules Explained (CommonJS vs ES Modules)

Understand Node.js modules like a pro (without losing your student sanity üòÖ)

Node.js modules are the building blocks of scalable JavaScript applications. They allow you to split code into reusable, organized pieces. Think of modules like your class notes during exams: organized, reusable, and life-saving. There are two main types of modules in Node.js: CommonJS (CJS) and ES Modules (ESM).

Students tip: Using modules is like having cheat sheets that actually help you instead of getting you caught üòé.

1. CommonJS Modules

CommonJS is Node.js's classic module system. You require() modules and module.exports your code. Modules are loaded synchronously, meaning Node waits for the module to load before moving forward.

Imagine borrowing notes from your friend during a group study session. You wait until your friend is done handing them over (synchronous), but it works reliably.

Syntax Example

// math.js
const add = (a, b) => a + b;
module.exports = { add };

// main.js
const { add } = require('./math');
console.log(add(2,3)); // 5

2. ES Modules (ESM)

ES Modules is the modern JavaScript module system. You use import and export. Modules are loaded asynchronously, making them more suitable for front-end and large-scale Node.js projects.

Think of ESM as accessing online lecture notes anytime. No need to wait for anyone. Fast, asynchronous, and flexible.

Syntax Example

// math.mjs
export const add = (a, b) => a + b;

// main.mjs
import { add } from './math.mjs';
console.log(add(2,3)); // 5

3. Key Differences Between CJS and ESM
