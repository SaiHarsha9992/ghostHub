1. What are Environment Variables?

Environment variables are dynamic values stored outside your code, letting your application behave differently depending on where it runs (development, staging, production).

  • PORT: The port your server listens on.
  • DB_URL: Database connection string.
  • API_KEY: Secret keys for APIs.

2. Setting up Environment Variables

You can use a .env file with the dotenv package:

// .env file
PORT=4000
DB_URL=mongodb://localhost:27017/mydb
API_KEY=12345abcde

// server.js
require('dotenv').config();
const express = require('express');
const app = express();

const PORT = process.env.PORT;
console.log('Database URL:', process.env.DB_URL);
console.log('API Key:', process.env.API_KEY);

app.listen(PORT, () => console.log(`Server running on port ${PORT}`));

Code Explanation:

  • require('dotenv').config(): Loads variables from .env into process.env.
  • process.env.PORT: Accesses the PORT variable.
  • process.env.DB_URL: Accesses the database URL.
  • process.env.API_KEY: Accesses your secret API key.

This approach keeps sensitive info out of your main code.

3. Why Use Environment Variables?

  • Security: Keep secrets out of code repositories.
  • Flexibility: Change settings without modifying code.
  • Portability: Easily move app between environments.

4. Configuration Management

Beyond environment variables, configuration management ensures your app has organized, structured settings. Common patterns:

  • config.js: Central file to manage all settings.
  • Multiple .env files: Separate variables for dev, test, production.
  • npm config: Store app-specific configuration using npm commands.

5. Example: Central Config File

// config.js
module.exports = {
    port: process.env.PORT || 3000,
    dbUrl: process.env.DB_URL || 'mongodb://localhost:27017/defaultdb',
    apiKey: process.env.API_KEY || 'defaultkey'
};

// server.js
const express = require('express');
const config = require('./config');
const app = express();

console.log('DB URL:', config.dbUrl);
console.log('API Key:', config.apiKey);

app.listen(config.port, () => console.log(`Server running on port ${config.port}`));

Code Explanation:

  • module.exports: Exports config object for reuse.
  • Each value checks process.env and uses a fallback if not set.
  • Keeps all settings in one place, improving maintainability.

Analogy: Like having a single notebook with all your class codes, passwords, and references neatly organized.
