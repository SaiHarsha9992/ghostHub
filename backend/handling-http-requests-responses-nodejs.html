Introduction

HTTP (HyperText Transfer Protocol) is the backbone of web communication. In Node.js, handling HTTP requests and responses is like managing a WhatsApp group:

  • Requests: Students send messages asking for notes, homework updates, or announcements.
  • Responses: You reply back with the info, images, or funny memes.

Mastering HTTP requests & responses allows you to control what data goes in and what comes out of your server.

HTTP Methods

HTTP provides standard methods for communication between client and server:

  • GET: Fetch data from the server.
  • POST: Send new data to the server.
  • PUT: Update existing data.
  • DELETE: Remove data from the server.
  • PATCH: Partially update existing data.

Student analogy: GET = check your results, POST = submit homework, PUT = edit assignment, DELETE = erase embarrassing chat.

1. Setting up a Basic HTTP Server in Node.js

Node.js has a built-in http module to handle requests and responses.

// Import HTTP module
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello, this is your Node.js server!');
});

server.listen(3000, () => {
    console.log('Server running at http://localhost:3000/');
});

Code Explanation:

  • require('http'): Imports the built-in Node.js HTTP module for creating a server.
  • http.createServer((req, res) => {...}): Creates an HTTP server.
  • req: Request object containing client request info (URL, headers, method, body).
  • res: Response object used to send data back to the client.
  • res.writeHead(200, {'Content-Type': 'text/plain'}): Sets HTTP status code (200 = OK) and response headers.
  • res.end('Hello...'): Ends the response and sends the message to the client.
  • server.listen(3000, () => {...}): Starts the server on port 3000 and runs the callback once the server is ready.

Start the server using node server.js. Your first Node HTTP server is ready to handle requests!

2. Understanding the Request Object

The req object contains all information about the clientâ€™s request:

  • req.url: The URL requested by the client.
  • req.method: HTTP method (GET, POST, etc.).
  • req.headers: Request headers like content-type or auth tokens.
  • req.body: Data sent by the client (available when using middleware like express.json()).

Think of it like reading a WhatsApp message: you know who sent it, what they want, and any extra info attached.

3. Understanding the Response Object

The res object allows you to send data back to the client:

  • res.status(code): Set HTTP status code.
  • res.setHeader(name, value): Set response headers.
  • res.write(data): Write data to the response.
  • res.end(data): End response and optionally send data.
  • res.json(data): Send JSON response (commonly used in APIs).

4. Handling GET & POST Requests

Example using Express.js for GET and POST:

const express = require('express');
const app = express();
app.use(express.json());

let messages = [];

// GET request
app.get('/messages', (req, res) => {
    res.json(messages);
});

// POST request
app.post('/messages', (req, res) => {
    const newMessage = req.body;
    messages.push(newMessage);
    res.status(201).json(newMessage);
});

app.listen(3000, () => console.log('Server running on http://localhost:3000'));

Code Explanation:

  • app.use(express.json()): Middleware to parse JSON request bodies.
  • let messages = [];: An array storing messages (simulating a simple database).
  • app.get('/messages', (req, res) => {...}): GET route to fetch all messages.
  • req: Request info (not used here).
  • res.json(messages): Sends the array as JSON to the client.
  • app.post('/messages', (req, res) => {...}): POST route to add a new message.
  • req.body: Contains the JSON data sent by the client.
  • messages.push(newMessage): Adds the new message to the array.
  • res.status(201).json(newMessage): Sends response with status 201 (Created).
  • app.listen(3000,...): Starts the server on port 3000.

Analogy: GET = checking all messages in the WhatsApp group, POST = sending a new message.

6. Advanced Topics

  • Query Parameters: Handle optional data in requests.
  • Route Parameters: Extract dynamic parts of URLs.
  • Error Handling: Send proper HTTP codes for errors.
  • Middleware: Process requests before route handling.
  • Streaming Data: Send or receive large data without loading it all in memory.
  • Asynchronous Handling: Handle multiple requests efficiently using async/await or callbacks.
