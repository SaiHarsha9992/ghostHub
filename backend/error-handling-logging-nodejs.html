1. Types of Errors

Node.js errors can be categorized to help you handle them more effectively:

Synchronous Errors

Happen in normal code execution line by line. Easily handled using try-catch.

try {
    const number = JSON.parse('123abc'); // Invalid JSON
} catch (error) {
    console.error('Synchronous error caught:', error.message);
}

Asynchronous Errors

Happen in callbacks, promises, or async/await. Require proper async error handling.

setTimeout(() => {
    try {
        throw new Error('Async error!');
    } catch (error) {
        console.error('Caught inside callback:', error.message);
    }
}, 1000);

Operational Errors

External issues like database unavailability, network failures, or file system errors.

const fs = require('fs');
fs.readFile('/wrong/path/file.txt', 'utf8', (err, data) => {
    if (err) console.error('Operational error:', err.message);
});

Programmer Errors

Bugs in your code like undefined variables, type errors, or logical mistakes.

function divide(a, b) {
    return a / b;
}
console.log(divide(5, x)); // ReferenceError: x is not defined

2. Basic Error Handling

Using try-catch for synchronous errors:

try {
    const data = JSON.parse('{"name":"John"}');
    console.log('Parsed data:', data);
} catch (error) {
    console.error('Error parsing JSON:', error.message);
}

Code Explanation:

  • try { ... }: Code that may throw an error.
  • catch (error) { ... }: Captures the error object if any error occurs.
  • error.message: Contains a human-readable error description.
  • console.error(...): Logs the error in red in your terminal.

3. Handling Asynchronous Errors

Async functions require try-catch or .catch() for promises:

async function fetchData() {
    try {
        // Assume fetch is available (e.g., node-fetch installed)
        const response = await fetch('https://api.example.com/data'); 
        const data = await response.json();
        console.log('Data:', data);
    } catch (error) {
        console.error('Error fetching data:', error.message);
    }
}

fetchData();

Code Explanation:

  • async function fetchData(): Declares an asynchronous function.
  • await fetch(...): Waits for the HTTP request to complete.
  • try-catch: Captures both synchronous and asynchronous errors inside async function.

4. Logging Best Practices

  • Use a Logging Library: Libraries like winston or pino for structured logs.
  • Separate Log Levels: info, warn, error, debug for clarity.
  • Include Timestamps: Helps trace issues in production.
  • Avoid Logging Sensitive Info: Keep passwords, API keys out of logs.
  • Centralized Logging: Tools like ELK stack, Graylog, or cloud logging for big apps.

5. Example: Using Winston for Logging

const winston = require('winston');

const logger = winston.createLogger({
    level: 'info',
    format: winston.format.combine(
        winston.format.timestamp(),
        winston.format.json()
    ),
    transports: [
        new winston.transports.Console(),
        new winston.transports.File({ filename: 'error.log', level: 'error' }),
        new winston.transports.File({ filename: 'combined.log' })
    ]
});

// Usage
logger.info('Server started successfully');
logger.error('Database connection failed');

Code Explanation:

  • winston.createLogger(): Creates a logger instance.
  • level: Minimum severity level to log.
  • format: Combines timestamp and JSON format for readable logs.
  • transports: Determines where logs are written (console, files).
  • logger.info(...) & logger.error(...): Log messages at different levels.

6. Advanced Topics

  • Global Error Handlers: process.on('uncaughtException') and process.on('unhandledRejection').
  • Structured Logging: Use JSON logs for easy parsing by log monitoring tools.
  • Monitoring & Alerts: Integrate with tools like Sentry, New Relic, or Datadog.
  • Contextual Logging: Include request ID, user info, and session context.

Proper error handling and logging are essential for building reliable and maintainable Node.js applications. Catch errors, log them smartly, and monitor your server to prevent chaos.
