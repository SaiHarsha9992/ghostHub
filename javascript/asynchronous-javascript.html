Asynchronous JavaScript

1. Introduction

JavaScript is single-threaded, meaning it executes code one line at a time. Asynchronous programming allows JavaScript to perform tasks like fetching data or waiting for timers without blocking the rest of the code. Three key ways to handle asynchronous code:

  • Callbacks
  • Promises
  • Async/Await

2. Callbacks

‚óè A callback is a function passed as an argument to another function to be executed later.

Example

function greet(name, callback) {
    console.log("Hello, " + name);
    callback();
}

function sayGoodbye() {
    console.log("Goodbye!");
}

greet("Durga", sayGoodbye);

‚óè Works, but nested callbacks can become hard to read (‚Äúcallback hell‚Äù).

3. Promises

‚óè A Promise represents a value that will be available now, later, or never.

‚óè States: pending, fulfilled, rejected.

Creating and Using a Promise

let fetchData = new Promise((resolve, reject) => {
    let success = true;
    if(success) {
        resolve("Data fetched successfully");
    } else {
        reject("Error fetching data");
    }
});

fetchData
    .then(result => console.log(result))
    .catch(error => console.log(error));

Advantages over Callbacks

  • Avoids nested callbacks
  • Easier error handling with .catch()
  • Can chain multiple asynchronous tasks

4. Async/Await

‚óè Introduced in ES8, allows writing asynchronous code like synchronous code.

‚óè Must be used with functions declared as async.

‚óè Use await to wait for a Promise to resolve.

Example

async function getData() {
    try {
        let result = await fetchData; // fetchData is a promise
        console.log(result);
    } catch(error) {
        console.log(error);
    }
}

getData();

Advantages

  • Cleaner, readable syntax
  • Handles Promises easily with try/catch
  • Avoids callback hell

6. Advantages ‚úÖ

  • Makes web apps responsive by handling tasks asynchronously
  • Promises and async/await simplify code structure
  • Error handling becomes more predictable
  • Works well with APIs, timers, and file operations

7. Disadvantages ‚ùå

  • Callbacks can lead to callback hell if nested deeply
  • Misusing async/await without try/catch can cause unhandled rejections
  • Understanding asynchronous flow can be confusing for beginners

8. Conclusion

  • Callbacks: basic way to handle async tasks but can become messy
  • Promises: cleaner, chainable, better error handling
  • Async/Await: modern syntax, easier to read, avoids callback hell
  • Mastering asynchronous JavaScript is essential for APIs, timers, and real-time applications
