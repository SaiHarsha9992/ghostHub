Template Literals, Optional Chaining, and Nullish Coalescing

1. Introduction

Modern JavaScript (ES6+) introduced several syntax improvements that make code cleaner, safer, and more readable. Three key features are:

  • **Template Literals** ‚Üí Easy string interpolation
  • **Optional Chaining (?.)** ‚Üí Safely access nested properties
  • **Nullish Coalescing (??)** ‚Üí Handle null or undefined values

2. Template Literals

‚óè Template literals use backticks (`) instead of quotes.

‚óè Allow string interpolation and multiline strings.

Example: String Interpolation

let name = "Durga";
let age = 20;

console.log(`My name is ${name} and I am ${age} years old.`);
// Output: My name is Durga and I am 20 years old.

Example: Multiline Strings

let message = `Hello Durga,
Welcome to JavaScript blogs!
Enjoy learning!`;

console.log(message);

3. Optional Chaining (?.)

‚óè Prevents errors when accessing nested properties of objects that may be undefined or null.

let user = {
    name: "Durga",
    address: { city: "Hyderabad" }
};

console.log(user.address?.city); // Hyderabad
console.log(user.contact?.phone); // undefined (does not throw error)

‚óè Useful when working with API responses where some fields may be missing.

4. Nullish Coalescing (??)

‚óè Returns the right-hand value only if the left-hand value is null or undefined.

‚óè Unlike ||, it does not treat 0 or an empty string as falsy.

let score = 0;
let defaultScore = score ?? 10;
console.log(defaultScore); // 0 (because 0 is not null or undefined)

let userScore;
let finalScore = userScore ?? 10;
console.log(finalScore); // 10 (because userScore is undefined)

5. Combining Optional Chaining and Nullish Coalescing

let user = {
    name: "Durga",
    preferences: null
};

let theme = user.preferences?.theme ?? "light";
console.log(theme); // light

‚óè Safely access nested properties without throwing errors and provide default values.

7. Advantages ‚úÖ

  • Template literals ‚Üí Cleaner string concatenation and multiline strings
  • Optional chaining ‚Üí Prevents runtime errors when accessing nested properties
  • Nullish coalescing ‚Üí Provides default values safely
  • Reduces the need for verbose checks and conditional statements

8. Disadvantages ‚ùå

  • May be confusing for beginners initially
  • Overuse of optional chaining can mask missing data bugs
  • Only works in modern browsers or environments supporting ES2020+
