CSS Combinators

CSS combinators are special characters that explain the relationship between HTML elements. They help you apply styles more precisely, depending on how elements are related in the HTML structure.

There are four main combinators in CSS:

1. Descendant Combinator (space)

The descendant combinator targets elements that are nested inside another element, at any level.

Syntax:

parent descendant {
  /* styles */
}

Example:

<div class="container">
  <p>This is a paragraph inside container.</p>
</div>

.container p {
  color: blue;
}

‚úÖ This changes the text color of all <p> elements inside .container to blue.

2. Child Combinator ( > )

The child combinator targets elements that are direct children of a parent.

Syntax:

parent > child {
  /* styles */
}

Example:

<div class="container">
  <p>Direct child paragraph</p>
  <div>
    <p>Nested paragraph</p>
  </div>
</div>

.container > p {
  color: red;
}

‚úÖ Only the direct <p> child of .container becomes red. The nested <p> inside another <div> is not affected.

3. Adjacent Sibling Combinator ( + )

The adjacent sibling combinator selects an element that is immediately after another element, sharing the same parent.

Syntax:

element1 + element2 {
  /* styles */
}

Example:

<h2>Heading</h2>
<p>This paragraph comes immediately after heading</p>
<p>This paragraph does not</p>

h2 + p {
  font-weight: bold;
}

‚úÖ Only the first <p> after <h2> becomes bold.

4. General Sibling Combinator ( ~ )

The general sibling combinator selects all elements that are siblings after a specific element.

Syntax:

element1 ~ element2 {
  /* styles */
}

Example:

<h2>Heading</h2>
<p>Paragraph 1</p>
<p>Paragraph 2</p>

h2 ~ p {
  color: green;
}

‚úÖ Both <p> elements after <h2> turn green.

Quick Summary Table

Combinator Symbol What it selects
Descendant (space) Any nested element inside parent
Child > Direct child only
Adjacent Sibling + Next element only
General Sibling ~ All siblings after

Why Use Combinators?

  • Makes CSS more specific without adding extra classes.
  • Helps in styling complex HTML structures efficiently.
  • Keeps HTML clean and semantic.
