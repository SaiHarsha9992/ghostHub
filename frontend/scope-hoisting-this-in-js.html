Scope, Hoisting, and the this Keyword

1. Introduction

When writing JavaScript, we must understand where variables can be accessed (Scope), how they behave before declaration (Hoisting), and how the special keyword this works.

These concepts control how your program runs behind the scenes.

2. Scope in JavaScript

Scope decides where a variable can be used.

(A) Global Scope

Variables declared outside any function/block are available everywhere.

üëâ Example:

let globalVar = "I am global";

function show() {
  console.log(globalVar); // Accessible here
}
show();
console.log(globalVar); // Accessible here too

(B) Local/Function Scope

Variables declared inside a function are only available there.

üëâ Example:

function greet() {
  let name = "Durga";
  console.log(name); // Works
}
greet();
// console.log(name); ‚ùå Error: not accessible outside

(C) Block Scope

Variables declared with let or const are limited to a { } block.

üëâ Example:

{
  let x = 10;
  const y = 20;
  console.log(x, y); // Works
}
// console.log(x, y); ‚ùå Error

üìå var does not have block scope, which is why let and const are recommended.

3. Hoisting in JavaScript

Hoisting is JavaScript‚Äôs default behavior of moving declarations to the top before execution.

üëâ Example with var:

console.log(a); // undefined (not error)
var a = 5;

üëâ Example with let/const:

// console.log(b); ‚ùå Error
let b = 10;

üìå Function declarations are also hoisted, so they can be called before they appear.

üëâ Example:

sayHello(); // Works fine

function sayHello() {
  console.log("Hello!");
}

4. The this Keyword

this refers to the object that is currently executing the function.

(A) In Global Scope

üëâ Example:

console.log(this); // Refers to window object (in browser)

(B) Inside an Object

üëâ Example:

let person = {
  name: "Durga",
  greet: function() {
    console.log("Hello, " + this.name);
  }
};
person.greet(); // Hello, Durga

(C) In a Function

üëâ Example:

function showThis() {
  console.log(this); // Refers to global object
}
showThis();

(D) In Arrow Functions

Arrow functions do not have their own this. They take it from the outer scope.

üëâ Example:

let obj = {
  name: "Durga",
  arrowFunc: () => {
    console.log(this.name); // undefined (does not refer to obj)
  }
};
obj.arrowFunc();

6. Conclusion

  • Scope ‚Üí Controls where variables can be accessed (Global, Function, Block).
  • Hoisting ‚Üí Moves variable/function declarations to the top before execution.
  • this ‚Üí Refers to the object that is executing the function.

üìå Mastering these concepts is essential for writing clean and bug-free JavaScript code.
