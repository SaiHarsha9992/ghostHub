Event Handling

1. Introduction

Event handling allows JavaScript to respond to user interactions such as clicks, typing, mouse movements, and form submissions. Key concepts include:

  • **Event listeners** ‚Üí Attach functions to events
  • **Event delegation** ‚Üí Handle events efficiently on multiple elements
  • **Event object** ‚Üí Provides details about the event

2. Adding Event Listeners

(A) Using addEventListener

let button = document.querySelector("button");
button.addEventListener("click", function() {
    alert("Button clicked!");
});

(B) Event Types

  • click ‚Üí Mouse click
  • mouseover ‚Üí Mouse over an element
  • keydown ‚Üí Key pressed
  • submit ‚Üí Form submitted
  • input ‚Üí Input value changed

3. Event Delegation

‚óè Delegation allows a parent element to handle events for its child elements, reducing multiple listeners.

let list = document.querySelector("ul");

list.addEventListener("click", function(e) {
    if(e.target.tagName === "LI") {
        console.log("Clicked on:", e.target.textContent);
    }
});

‚óè Useful for dynamic elements added after page load.

4. The Event Object

‚óè Automatically passed to the event handler as a parameter (commonly e or event). Contains information about the event and the target element.

Common Properties

let button = document.querySelector("button");

button.addEventListener("click", function(e) {
    console.log("Event type:", e.type); // click
    console.log("Event target:", e.target); // button element
});

Common Methods

  • e.preventDefault() ‚Üí Prevent default browser action
  • e.stopPropagation() ‚Üí Stop event from bubbling up
  • e.stopImmediatePropagation() ‚Üí Stop other listeners on the same element

5. Mini-Practice Tasks üìù

  • Add a click event to a button and log a message.
  • Add a keydown event on an input to log the key pressed.
  • Use event delegation on a list of items to log which item is clicked.
  • Prevent form submission using preventDefault() and show an alert instead.
  • Stop an event from bubbling using stopPropagation() in nested elements.

6. Advantages ‚úÖ

  • Makes web pages interactive and responsive
  • Delegation improves performance on multiple elements
  • Event object provides detailed information for dynamic handling
  • Works with all standard DOM events

7. Disadvantages ‚ùå

  • Improper use of delegation can lead to unexpected behavior
  • Forgetting preventDefault() may trigger default browser actions
  • Adding too many listeners can impact performance

8. Conclusion

  • Use addEventListener to attach events to elements
  • Event delegation allows efficient handling of child elements
  • The event object provides key details about the interaction
  • Mastering events is essential for interactive and dynamic web applications
