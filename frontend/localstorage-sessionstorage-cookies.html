LocalStorage, SessionStorage, and Cookies

1. Introduction

Web storage allows JavaScript to store data in the browser. This is useful for saving user preferences, session info, and temporary data. There are three main storage options:

  • LocalStorage ‚Üí Stores data permanently until cleared
  • SessionStorage ‚Üí Stores data for the current session only
  • Cookies ‚Üí Small data stored by the browser, sent to the server with requests

2. LocalStorage

‚óè Stores data as key-value pairs in the browser

‚óè Persists even after closing the browser

Basic Methods

// Set item
localStorage.setItem("username", "Durga");

// Get item
let name = localStorage.getItem("username");
console.log(name); // Durga

// Remove item
localStorage.removeItem("username");

// Clear all items
localStorage.clear();

3. SessionStorage

‚óè Stores data for the duration of the browser tab session

‚óè Cleared automatically when the tab is closed

Basic Methods

// Set item
sessionStorage.setItem("sessionName", "TestUser");

// Get item
console.log(sessionStorage.getItem("sessionName"));

// Remove item
sessionStorage.removeItem("sessionName");

// Clear all items
sessionStorage.clear();

4. Cookies

‚óè Small pieces of data stored by the browser

‚óè Sent with every HTTP request to the server

‚óè Can have an expiry date

Creating and Reading Cookies

// Set a cookie
document.cookie = "username=Durga; expires=Fri, 31 Dec 2025 23:59:59 GMT; path=/";

// Read all cookies
console.log(document.cookie);

‚óè Note: Cookies are limited in size (~4KB) and slower than web storage.

5. Differences Between Storage Options

Feature LocalStorage SessionStorage Cookies
Lifetime Permanent Current session Set expiry date / default session
Data sent to server No No Yes
Storage size ~5-10 MB ~5-10 MB ~4 KB
Scope Across tabs Current tab only Across tabs depending on path/domain

7. Advantages ‚úÖ

  • Easy to store and retrieve data in the browser
  • Reduces server requests by storing data locally
  • Useful for offline applications
  • Simple API (setItem, getItem, removeItem)

8. Disadvantages ‚ùå

  • Data is not secure (anyone with browser access can view it)
  • Cookies are limited in size and sent with every request
  • LocalStorage and SessionStorage are synchronous, can block the main thread for large data
  • Not suitable for sensitive information

9. Conclusion

  • LocalStorage ‚Üí Persistent, client-side storage
  • SessionStorage ‚Üí Temporary storage for the session
  • Cookies ‚Üí Small data sent to the server with requests
  • Choosing the right storage depends on data size, lifetime, and sensitivity
  • Mastering web storage is essential for modern web applications, user preferences, and offline functionality
