Debugging and Error Handling in JavaScript

1. Introduction

Errors are inevitable in programming. JavaScript provides tools to detect, debug, and handle errors gracefully. Proper debugging and error handling improve code reliability and user experience.

2. Types of Errors in JavaScript

Syntax Errors

‚Üí Mistakes in code structure

console.log("Hello World" // Missing closing parenthesis

Runtime Errors

‚Üí Occur during execution

let num;
console.log(num.toFixed(2)); // Error: Cannot read property 'toFixed' of undefined

Logical Errors

‚Üí Code runs but produces the wrong output

let total = 10 + "5"; // Output: "105" instead of 15

3. Debugging Techniques

(A) Using console.log

‚óè Most common method to trace values and execution.

let x = 10;
console.log("Value of x:", x);

(B) Using console.error

‚óè Shows errors clearly in the console.

console.error("Something went wrong!");

(C) Using console.table

‚óè Displays arrays or objects in table format for easier inspection.

let users = [
    {name: "Durga", age: 20},
    {name: "Lakshmi", age: 22}
];
console.table(users);

(D) Browser Debugger

‚óè Use breakpoints, step-over, step-into, and inspect variables directly in browser dev tools.

4. Error Handling with try...catch

‚óè try ‚Üí Code that may throw an error

‚óè catch ‚Üí Handle the error without stopping execution

‚óè finally ‚Üí Code that runs regardless of the error

try {
    let result = riskyOperation(); // May throw an error
    console.log(result);
} catch (error) {
    console.log("Error occurred:", error.message);
} finally {
    console.log("Execution finished");
}

5. Throwing Custom Errors

‚óè Use throw to create custom errors.

function checkAge(age) {
    if(age < 18) {
        throw new Error("You must be 18 or older");
    } else {
        console.log("Access granted");
    }
}

try {
    checkAge(16);
} catch (error) {
    console.log(error.message);
}

  • Use console.table to display an array of objects.

  • Debug a function using console.log to find a logical error.
  • Use the browser debugger to step through a simple loop and inspect variables.

7. Advantages ‚úÖ

  • Helps identify and fix bugs quickly
  • Prevents the program from crashing unexpectedly
  • Provides meaningful error messages to developers
  • Improves code reliability and user experience

8. Disadvantages ‚ùå

  • Overusing try...catch for normal control flow can slow down performance
  • Console debugging may be less effective for complex applications
  • Logical errors still require careful inspection, cannot be caught automatically

9. Conclusion

  • JavaScript errors: syntax, runtime, logical
  • Debugging tools: console.log, console.error, console.table, browser debugger
  • Error handling: try...catch, finally, and custom throw
  • Mastering debugging and error handling is essential for writing robust, maintainable JavaScript code
