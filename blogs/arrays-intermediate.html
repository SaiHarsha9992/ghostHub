Problem 1: Find the Maximum and Minimum

ðŸ‘‰ Strategy: Traverse once, keep track of current max and min.

C++: Find Max and Min

#include 
using namespace std;
int main() {
    int arr[] = {12, 5, 9, 34, 7};
    int n = 5, maxi = arr[0], mini = arr[0];
    for(int i = 1; i < n; i++) {
        maxi = max(maxi, arr[i]);
        mini = min(mini, arr[i]);
    }
    cout << "Max = " << maxi << ", Min = " << mini;
    return 0;
}

Problem 2: Reverse an Array

ðŸ‘‰ Strategy: Swap elements from start and end moving inward.

Python: Reverse Array

arr = [1, 2, 3, 4, 5]
i, j = 0, len(arr)-1
while i < j:
    arr[i], arr[j] = arr[j], arr[i]
    i += 1
    j -= 1
print("Reversed:", arr)

Problem 3: Find the Second Largest Element

ðŸ‘‰ Strategy: Keep two variables: largest and second largest.

Java: Second Largest Element

public class SecondLargest {
    public static void main(String[] args) {
        int arr[] = {10, 20, 5, 8, 30};
        int first = Integer.MIN_VALUE, second = Integer.MIN_VALUE;
        for (int num : arr) {
            if (num > first) {
                second = first;
                first = num;
            } else if (num > second && num != first) {
                second = num;
            }
        }
        System.out.println("Second Largest = " + second);
    }
}

2. Basic Strategies for Array Problem-Solving

Here are some battle-tested strategies you should follow when solving array problems:

  • Understand the problem clearly

    Identify input size (n) and constraints. Example: n <= 10^5 â†’ O(nÂ²) solution may not work.

  • Start with brute force

    Donâ€™t rush to optimize. First, write the simplest solution. Example: To check duplicates â†’ nested loops.

  • Optimize step by step

    Replace brute force with sorting or hash maps if possible. Example: Duplicates â†’ sort array â†’ check neighbors.

  • Use two-pointer technique

    Useful for problems like reversing arrays, pair sum, sliding window.

  • Think about prefix sums / cumulative arrays

    Great for subarray sum queries.

  • Draw examples

    Always try with small inputs manually before coding.

  • Check edge cases

    Empty array, single element, all equal elements, very large inputs.
