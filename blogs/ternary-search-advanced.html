Ternary Search

Part 2: Advanced Ternary Search

1. When to Use Ternary Search?

Ternary search is a powerful tool for more than just simple array searching. It is especially useful for:

  • Finding the maximum or minimum in **unimodal functions** (a common pattern in competitive programming).
  • Situations where binary search comparisons are computationally costly.
  • Large data sets where dividing the search space by three can slightly reduce recursion depth.

2. Real-Life Advanced Example

Scenario: Optimizing the profit function in a business, where profit depends on a variable like price. A profit function is often unimodal; it increases as you raise the price (because of a higher margin) and then decreases after a certain point (because fewer customers are willing to buy the product).

Ternary search helps find the **price** that gives the maximum profit efficiently by repeatedly narrowing the price range.

3. Ternary Search for Unimodal Functions

The core logic of ternary search can be applied to continuous functions. The goal is to narrow down an interval $[l, r]$ to find the point where the function reaches its maximum or minimum. This iterative process continues until the interval is sufficiently small, determined by a small `eps` (epsilon) value.

Python Example

def f(x):
    # A unimodal function that increases then decreases
    return -(x - 2)**2 + 4

def ternary_search(l, r, eps=1e-6):
    while r - l > eps:
        mid1 = l + (r - l) / 3
        mid2 = r - (r - l) / 3

        if f(mid1) < f(mid2):
            l = mid1
        else:
            r = mid2
    return (l + r) / 2

max_x = ternary_search(0, 5)
print("Maximum at x =", max_x)
print("f(x) =", f(max_x))

4. Key Points for Advanced Learners

  • Use for continuous search problems (functions) where the solution space is continuous rather than discrete.
  • Accuracy vs. Iterations: A smaller eps (epsilon) value gives more precise results but requires more iterations.
  • Can be applied in AI hyperparameter tuning, economics, and engineering optimization problems.
